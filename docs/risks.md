# Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Application deadline or form changes | Submission package can go stale | Re-check official application page before final submit and update field map. |
| Missing Loom video | Application cannot be completed | Record 2 to 5 minute video using `docs/pitch.md` outline. |
| No deployed URL | Demo credibility lower | Deploy static folder to Vercel, Netlify or GitHub Pages; add URL to answers template. |
| Product too broad | Judges may see scope risk | Keep wedge narrow: hurricane logistics first, then expand to ports, shelters and suppliers. |
| Sensitive operational data | Trust and compliance risk | Demo uses public/synthetic data; pilots require agreements, RBAC, encryption and audit logs. |
| AI hallucinated actions | Operational safety risk | Require evidence links, confidence, human approval gates and clear uncertainty states. |
| Partner adoption friction | Sales cycle risk | Start with manual paste/CSV/email intake before deeper integrations. |
| Compute dependency | Build risk if resources are limited | Static demo and deterministic workflows run without GPU; use H200 access for model experiments only. |
| Direct form submit failure | Application may not transmit | Use official browser form as fallback and email apply@futurecaribbean.com if needed. |
