# Three Week Build Plan

## Week 1 - T0 wedge, data and operator truth

- Day 1: lock hurricane logistics as the first paid wedge and define the T0
  metric: baseline minutes, compressed minutes and economic drag.
- Day 2: map official/public data sources: weather advisories, port/airport
  status, open map layers and synthetic partner inventories.
- Day 3: create task schema: incident, request, owner, ETA, evidence, risk,
  approval state and public/private notes.
- Day 4: build agent workflow for intake, prioritization, task generation,
  incident ledger, economic scoring and situation brief drafting.
- Day 5: run two operator interviews or expert reviews and convert feedback into
  product acceptance tests.

## Week 2 - Operational network and paid pilot readiness

- Day 6: implement cockpit prototype with T0 score, live incident board and
  visible agent trace.
- Day 7: add partner channel ingestion for email/manual paste/CSV as first
  low-friction connectors.
- Day 8: add evidence ledger and human approval gates for any external message.
- Day 9: add exportable briefs and buyer-specific deal rooms for emergency ops,
  hotels, ports, suppliers and public stakeholders.
- Day 10: run scenario tests for hurricane logistics, medical supply delay and
  shelter surge.

## Week 3 - Deployment, traction and investor-grade pitch

- Day 11: deploy hosted pilot with observability, backups and environment
  separation.
- Day 12: add role-based views for operator, partner and viewer.
- Day 13: create pilot onboarding kit and data handling policy.
- Day 14: gather operator feedback, advisor quotes or paid-pilot target asks.
- Day 15: final demo video, investor pitch, submission package and production
  deployment checklist.

## Success metrics

- Reduce coordination brief creation from hours to minutes.
- Show at least one T0 compression model with economic drag and pilot pricing.
- Produce action boards with owner and ETA coverage above 90%.
- Keep every externally shared brief linked to evidence and approval state.
- Secure at least one Caribbean operator/advisor validation during the build.
